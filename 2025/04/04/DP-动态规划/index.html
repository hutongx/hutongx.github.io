DP 动态规划

Posted on 2025-04-04 In Leetcode , 动态规划 Views:

下面给出一个高度抽象、通用的动态规划（DP）解题模板，全部采用 Java 语言实现。该模板不依赖于具体题目，而是按照动态规划的基本思路构建：问题拆分、状态定义、状态转移、初始化、答案提取，以及必要时的空间优化。可以根据实际题目的特点，在相应的位置替换或扩展代码逻辑。

一、动态规划解题通用步骤

解题思路

  1. 问题分析与拆分子问题

    • 分析题意，判断问题能否拆分为多个子问题，并检查是否存在重叠子问题。

  2. 状态定义

    • 明确用一个或多个 dp 数组/矩阵来描述子问题的最优解。
    • 例如：
      • 对于一维数组问题，通常定义 dp[i] 表示到第 i 个元素的最优状态；
      • 对于字符串问题，定义 dp[i][j] 表示两个字符串前 i 和 j 个字符的最优解；
      • 对于二维矩阵问题，定义 dp[i][j] 表示从起点到 (i, j) 的最优路径值（或计数等）。

  3. 设计状态转移方程

    • 找出子问题之间的联系，用前面求解的状态推导当前状态。
    • 注意讨论转移时的所有可能情况，例如字符匹配与不匹配、不同方向的选择等。

  4. 初始化与边界条件

    • 给 dp 数组或矩阵中最小规模的子问题赋初值，避免越界问题。
    • 比如：对于字符串 DP，常见的边界是 dp[0][j] = 0 和 dp[i][0] = 0；对于矩阵 DP，则通常先初始化第一行和第一列。

  5. 答案提取

    • 根据题目要求，从 dp 数组或矩阵中提取最终答案。
    • 可能是直接返回 dp[n]、dp[m][n]，或者遍历 dp 得到全局最值。

  6. 代码优化

    • 如果状态转移只依赖于部分状态，可以考虑空间压缩（如滚动数组）来降低空间复杂度。

代码模版

模板说明

  1. 抽象与通用性

    • 每个方法均遵循动态规划的基本步骤：
      ① 定义状态（例如：dp 数组/矩阵）；
      ② 初始化基本子问题；
      ③ 通过循环构造状态转移；
      ④ 最后提取最终答案。
    • 方法中的细节（如初始值、转移函数、合并函数）都采用抽象形式，需要根据实际题目修改。

  2. 扩展与应用

    • 若问题为字符串或二维矩阵问题，可直接参考相应模板；
    • 对于其他类型的 DP 问题，只需调整状态定义和转移逻辑即可；
    • 模板中的注释明确说明了每个部分的作用，便于迁移至具体题目。

  3. 核心代码模式

    • 保持代码简洁、模块化，便于在题中快速复用和调试；
    • 空间优化、自顶向下（递归+记忆化）等技巧也可在此基础上扩展。

三、进阶技巧与注意事项

  • 空间优化

    • 如果状态转移仅依赖于上一行或前一个状态，可以将二维 dp 数组压缩为一维数组，节省空间。

  • 自顶向下与自底向上

    • 根据题目规模选择递归+记忆化（自顶向下）或迭代（自底向上）的实现方式，二者效果相近，重在清晰表达状态转移逻辑。

  • 边界处理

    • 处理好初始化与边界条件，避免在状态转移时出现数组越界或错误状态。

  • 核心代码模式

    • 保持代码简洁和模块化，注释清晰的状态定义和转移逻辑有助于复用和快速调试。
